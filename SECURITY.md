# Security Policy

## Reporting a Vulnerability

If you discover a security issue in any of the projects in this repository (such as in the PhysicsCalc HTML/JS application), please report it privately to the maintainer.

- Preferred: Open a private security advisory on GitHub (if available for this repo).
- Or email the maintainer directly (see profile for contact).

Do **not** open public issues for security vulnerabilities.

## Repository Policy

This repository is maintained as a **personal, read-only archive** of projects created in collaboration with Grok (xAI).

- **No public contributions are accepted.** 
- Pull requests and issues from external parties will not be reviewed or merged.
- The repository is intended **only for reference and direct file sharing by the owner**.
- **Do not fork this repository** for the purpose of running or distributing modified versions of the included applications.
- **Do not run modified copies** of any HTML/JS files (e.g., PhysicsCalc `index.html`) obtained from forks, third-party downloads, or untrusted sources. These files contain executable code and could be tampered with to include malware.
- **Strict preservation rule:** Future updates (by owner + Grok only) must **never delete or alter existing projects** (e.g., do not remove or change the `physics-calculator/` directory when adding new content). New projects are added as additional subdirectories only, and the root README.md is updated by *appending* a description of the new addition (no rewriting or removal of prior project entries).

## Recommendations for Users

- Always obtain the original `.html` files directly from the owner (e.g., via direct download or trusted channel).
- For the PhysicsCalc tool and similar single-file web apps, the safest way to share is to distribute the original file itself, not links to the repository or forks.
- The owner strongly recommends keeping this repository set to **Private** visibility to prevent unauthorized access, forking, or scraping of the source.

## Why This Policy?

Single-file HTML + JavaScript applications are convenient for distribution but are inherently executable content. To protect users from supply-chain style attacks via modified forks or tampered files, this repo is deliberately not open for external edits or public forking.

All projects here were built iteratively with Grok using the HelixCore platform for governance, safety, and closed-loop development. The focus is on quality and safety, not open-source collaboration.

## Contact

For any security concerns or questions about this policy, contact the maintainer via their GitHub profile.

Thank you for respecting these guidelines.
